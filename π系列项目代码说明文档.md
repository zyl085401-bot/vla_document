1. 加载数据集【运行训练程序会自动加载数据集】

原项目通过`LeRobotDataset`来加载数据，如果在/root/.cache/huggingface(docker路径，Ubuntu路径：\~/.cache/huggingface)下没有查找到相应的数据集，会触发在线下载，下载后会保存到上面的默认路径中。

```python
def create_torch_dataset(
    data_config: _config.DataConfig, action_horizon: int, model_config: _model.BaseModelConfig
) -> Dataset:
    """Create a dataset for training."""
    repo_id = data_config.repo_id
    if repo_id is None:
        raise ValueError("Repo ID is not set. Cannot create dataset.")
    if repo_id == "fake":
        return FakeDataset(model_config, num_samples=1024)

    dataset_meta = lerobot_dataset.LeRobotDatasetMetadata(repo_id)
    dataset = lerobot_dataset.LeRobotDataset(
        data_config.repo_id,
        delta_timestamps={
            key: [t / dataset_meta.fps for t in range(action_horizon)] for key in data_config.action_sequence_keys
        },
    )

    if data_config.prompt_from_task:
        dataset = TransformedDataset(dataset, [_transforms.PromptFromLeRobotTask(dataset_meta.tasks)])

    return dataset
```

## demo示例

1. aloha\_sim

(1)构建Docker环境【仿真环境 - examples/aloha\_sim/Dockerfile】

```plain&#x20;text
 docker compose -f examples/aloha_sim/compose.yml up --build
```

**注意：这个Docker文件也经过修改。**

Docker会构建aloha\_sim和openpi\_server（上一步已构建）镜像，启动aloha\_sim\_openpi\_server-1和aloha\_sim-runtime-1 两个容器。

同时启动下面两个服务，两个服务之间通过websocket交换数据。

```yaml
# demo示例
#compose.yml 包含两个服务：
services:
  runtime:          # 机器人运行时（客户端）
    - 采集观察
    - 发送到服务器
    - 执行动作
  
  openpi_server:    # 推理服务器
    - 加载模型
    - 接收观察
    - 返回动作
```

（2）在aloha\_sim\_openpi\_server-1启动推理服务器

&#x20;       代码会在线加载aloha\_sim的checkpoint，可按照之前的步骤来下载。

```plain&#x20;text
export SERVER_ARGS="--env ALOHA --default_prompt='take the toast out of the toaster'"
uv run scripts/serve_policy.py $SERVER_ARGS   ### 服务器中运行
```

（3）在aloha\_sim-runtime-1 启动仿真环境

```plain&#x20;text
source /.venv/bin/activate && python examples/aloha_sim/main.py ### 仿真环境中运行
```
