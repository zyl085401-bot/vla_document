# 客户端数据流程【推理】【openpi/examples/aloha\_sim】

### 1. 文件依赖关系

main.py -> runtime.py -> env.py

### 2. 函数调佣次序

```plain&#x20;text
Runtime.init -> Runtime._run_episode -> AlohaSimEnvironment.get_observation -> PolicyAgent.get_action(observation) AlohaSimEnvironment.apply_action -> AlohaSimEnvironment._convert_observation
```

> * Runtime.\_run\_episode 【环境、智能体、订阅者初始化】
>
> * Runtime.\_run\_episode 中执行循环，每次循环进行一个 step：
>
>   1. \_environment.get\_observation() -> 获取观测信息
>
>   2. \_agent.get\_action(observation)
>
>      1. agent 就是接收观察、产出动作的模块；在示例里它包装了远程策略客户端
>
>   3. \_environment.apply\_action(action) -> 动作应用 & 更新观察信息

### 3. 策略调佣说明【agent.get\_action】

```plain&#x20;text
PolicyAgent.infer(observation) -> ActionChunkBroker.infer -> WebsocketClientPolicy
```

> agent 是“决策者”抽象，负责把当前观测变成下一步动作。在本项目里：
>
> * 抽象接口：openpi\_client.runtime.agent.Agent 定义 get\_action(observation)&#x20;
>
> * 具体实现：PolicyAgent，内部持有一个 policy，在 get\_action 里直接调用 policy.infer(observation)。
>
> * 在 Aloha 示例中，这个 policy 是 ActionChunkBroker(WebsocketClientPolicy(...))，即通过 WebSocket 去远程策略服务器拿动作，并做动作序列缓存。

### 4. **观测数据结构说明**

*self*.\_last\_obs = *self*.\_convert\_observation(gym\_obs) = {"state": *gym\_obs*\["agent\_pos"], "images": {"cam\_high": img},}

> state: 是 14 维数组，包含左臂 6 个关节、左夹爪 1 个、右臂 6 个关节、右夹爪 1 个
>
> images: 图像字典，key值指明图像名称，图像shape是 *\[C, H, W]*。

### 5. **返回结果说明**【\_agent.get\_action】

> 返回结果是：14 维数组，包含左臂 6 个关节、左夹爪 1 个、右臂 6 个关节、右夹爪 1 个

### 6. 客户端设计建议

> **推理时只需要更换env模块为自己的即可&#x20;**

# 服务器端匹配

服务器端通过 openpi/src/openpi/policies 下的策略进行数据转换和模型选择。可使用 SERVER\_ARGS 指定策略/数据。

```json
export SERVER_ARGS="--env ALOHA_SIM"
```

# 注意事项

1. 远程访问服务器需要配置客户端的IP

2. WSL中运行服务器需要配置防火墙规则或者关闭防火墙，且WSL使用镜像模式以及Docker使用host网络模式

```plain&#x20;text
netsh advfirewall firewall add rule ` name="WSL2 WebSocket 8000" ` dir=in action=allow protocol=TCP localport=8000
```

```plain&#x20;text
Set-NetFirewallProfile -Profile Public,Private -Enabled False
```
